安装 Oh My Browser

安装 CLI + 扩展

curl -fsSL https://api.omb.org.cn/install/install.sh | sh

安装器自动探测本机已装的 AI 客户端（Claude Code、Cursor、Codex、Windsurf、Trae……），为每个配置 MCP，并给你 Chrome 网上应用店链接装 Lite 扩展。

连接账户

在 omb.org.cn → Token 管理 创建 Token，然后：

omb connect <你的 token>

加载浏览器扩展

Chrome 网上应用店上线后，安装器末尾会给你直链。在此之前，安装器下载 unpacked 版，通过 chrome://extensions → 加载已解压的扩展程序 加载即可。

Full 版 —— 我要装吗？

大多数人装 Lite 就够了。只有当 agent 明确告诉你"需要 Full"（报错 EDITION_REQUIRES_FULL）或者你本来就是用 unpacked 方式开发，才装 Full。

一行命令安装

curl -fsSL https://omb.org.cn/install.sh | sh -s -- --edition=full

安装器下载校验过的 zip，解压到 ~/.omb/extensions/full/unpacked/，清除 macOS 隔离标记，然后打开 chrome://extensions。最后一步你手动做（见下）。

Full 是 unpacked 扩展。Chrome 会一直显示"开发者模式"横幅，每次启动都会弹窗提示。这是正常的 —— Chrome 网上应用店政策禁止 chrome.debugger，而这恰恰是 Full 做手势合成 + body 抓取用的 API。

手动加载步骤（install.sh 已经自动完成 1–3 步）

打开扩展页面。如果安装器没自动开：chrome://extensions。
打开开发者模式。右上角的开关。
点"加载已解压的扩展程序"。选 ~/.omb/extensions/full/unpacked/ 目录。
验证。跑 omb doctor —— 会确认 Full 装好、session 已连。

为什么分两个版本？

短答

Chrome 网上应用店的政策实际上禁止 chrome.debugger 和 chrome.webRequestBlocking（仅开发者工具类扩展可用）。我们不能走商店发 Full，也不会想办法混进去 —— 一旦被发现整个 listing 都会下架。

Lite 满足 80% 用户：读、搜索、点击、填表、执行 JS。过审、自动更新。Full 给剩下 20% 需要可信事件合成或完整 body 抓取的人，代价是忍受"未打包扩展"那条横幅。

技术上这个分割靠 build 时强制：esbuild 把 @omb/debugger 的 alias 在 Lite 切到一个会抛异常的 stub，所以 Lite 的 bundle 物理上不含 chrome.debugger 字符串。细节见 docs/edition-capabilities.md。

配置 MCP 客户端

Cursor：安装器直接写 ~/.cursor/mcp.json。Claude Desktop：把 dashboard 上的 JSON 粘到 ~/Library/Application Support/Claude/claude_desktop_config.json（macOS）或 %APPDATA%\Claude\（Windows）。

完整支持的客户端列表见 首页。
