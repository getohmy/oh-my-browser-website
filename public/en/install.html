Install Oh My Browser

Install the CLI + extension

curl -fsSL https://api.omb.org.cn/install/install.sh | sh

The installer picks up your detected AI clients (Claude Code, Cursor, Codex, Windsurf, Trae, …), configures MCP for each, and hands you a Chrome Web Store link for the Lite extension.

Connect your account

Create a token at omb.org.cn → Token Management, then:

omb connect <your-token>

Load the browser extension

Once the Chrome Web Store listing is live, the installer summary hands you a direct link. Until then, the installer downloads an unpacked build you can load via chrome://extensions → Load unpacked.

Full edition — when do I need it?

Most people want Lite. Install Full only if an agent tells you it needs it (you'll see an error with code EDITION_REQUIRES_FULL), or if you're a developer running an unpacked build anyway.

One-command install

curl -fsSL https://omb.org.cn/install.sh | sh -s -- --edition=full

The installer downloads a verified zip, unpacks it to ~/.omb/extensions/full/unpacked/, clears macOS quarantine flags, and opens chrome://extensions. You do the last step manually (see below).

Full is an unpacked extension. Chrome will show a permanent "Developer mode" banner and warn you on every relaunch. This is expected — Chrome Web Store policy does not allow extensions with chrome.debugger, which is what Full uses for gesture synthesis and body capture.

Manual load steps (after install.sh does steps 1–3 automatically)

Open the Extensions page. If the installer didn't open it for you: chrome://extensions.
Turn on Developer mode. Toggle in the top-right corner.
Click "Load unpacked". Navigate to ~/.omb/extensions/full/unpacked/ and choose that folder.
Verify. Run omb doctor — it confirms the Full build loaded and a session is live.

Why two editions?

The short answer

Chrome Web Store policy effectively forbids chrome.debugger and chrome.webRequestBlocking for anything except narrow developer-tools use cases. We can't ship Full through the Web Store, and we won't try to smuggle those APIs past review — that breaks the store listing for everyone.

Lite is what 80% of users need: read, search, click, fill, JS evaluation. It passes audit and auto-updates. Full is for the remaining 20% who need trusted-event synthesis or body capture, and who accept the "unpacked extension" warning in exchange.

Technically the split is enforced at build time: esbuild aliases @omb/debugger to a throwing stub on Lite, so Lite bundles physically cannot contain the string chrome.debugger. See docs/edition-capabilities.md.

Configure an MCP client

For Cursor, the installer writes ~/.cursor/mcp.json directly. For Claude Desktop, copy the JSON from the dashboard into ~/Library/Application Support/Claude/claude_desktop_config.json (macOS) or %APPDATA%\Claude\ (Windows).

See the homepage for the full list of supported clients.
