Privacy Policy — Summary

Canonical document: the full, authoritative text lives at store-assets/chrome-web-store/privacy-policy.md in the public source mirror. This page is an executive summary; the markdown is binding.

What we do NOT collect

  • URLs you visit
  • Page content, forms, or anything your agent reads
  • Cookies, session tokens, or credentials
  • Browsing history, bookmarks, or clipboard
  • Your IP address in identifiable form (we only store hashed prefixes for rate-limit audits)

What we do collect

  • Account data: email + OAuth identifier when you sign up. Used to provide service; deleted when you delete your account.
  • Anonymous telemetry: task type + success/failure + duration + hashed domain + error code (from a fixed enum). 90-day retention. No URLs, no page content.
  • Error reports: JavaScript exceptions in the extension / dashboard, with PII stripped via a beforeSend hook. Sent to Sentry.
  • Request metrics: aggregate dashboards of success rate + p95 latency, grouped by hashed domain only.
  • Site Memory: structural CSS selectors that worked, per domain, stripped of any identifying content. See the canonical doc for the full disclosure including the omb_memory(url) peek endpoint — it never returns selector strings or page content.
  • Browser fingerprint: SHA-256 hash of platform + UA + resolution + color depth + timezone + language + CPU threads. Used to route agent tasks to the right browser when you use OMB on multiple machines. Treated as pseudonymous personal data under GDPR.

What we never do

  • Sell or share data with third parties (the only third parties are infrastructure: Cloudflare / Sentry).
  • Profile users across services.
  • Fingerprint before you install and log in.

Your rights

Delete your account from the dashboard user menu. Account deletion cascades to devices, telemetry rows tied to your user_id, and private-layer Site Memory entries.

Contact

Security: security@omb.org.cn · Privacy: privacy@omb.org.cn
